# 设备拓扑（网关-子设备）

Simple IoT 支持设备间的父子拓扑关系，适用于网关下挂子设备的场景（如 Modbus 网关 + 多个从站）。

## 数据模型

`tb_device` 表新增 `parent_id` 字段，指向父设备的 ID。一个设备只能有一个父设备，但一个父设备可以有多个子设备。

## API

### 查询子设备列表

```
GET /device/{parentId}/children
```

**响应示例：**

```json
{
  "code": 200,
  "data": [
    {
      "id": 2,
      "deviceName": "Modbus 从站 1",
      "deviceKey": "modbus-slave-01",
      "parentId": 1
    }
  ]
}
```

## 前端操作

1. 进入「设备管理」页面
2. 点击某个网关设备的「详情」
3. 切换到「子设备」Tab，查看该网关下所有子设备

## 设置拓扑关系

目前通过设备编辑页面的 `parentId` 字段设置。后续版本将支持在子设备详情页直接绑定父设备。

## 典型场景

- **Modbus 网关**：网关设备通过 Modbus TCP 轮询多个从站，每个从站作为子设备
- **LoRa 网关**：网关设备接收多个 LoRa 节点的数据
- **边缘网关**：边缘计算网关代理多个本地设备
